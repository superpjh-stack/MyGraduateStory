## 1. 디자인 철학

### 1.1 핵심 디자인 원칙

```
"1년의 여정을 담는 그릇은, 따뜻하고 고요해야 한다."
```

| 원칙 | 설명 | 적용 방법 |
|------|------|-----------|
| **Warmth (따뜻함)** | 차가운 대시보드가 아닌, 일기장을 펼치는 느낌 | 따뜻한 컬러 팔레트, 부드러운 곡선, 자연 메타포 |
| **Calm (고요함)** | 바쁜 직장인에게 시각적 소음을 주지 않는다 | 여백 활용, 최소한의 장식, 깔끔한 타이포그래피 |
| **Growth (성장)** | 시간의 흐름과 성장을 시각적으로 체감 | 점진적 변화 애니메이션, 누적 시각화 |
| **Story (이야기)** | 데이터가 아니라 이야기를 전달한다 | 서사적 구성, 감정 흐름, 챕터 구조 |

### 1.2 디자인 메타포: "사계절의 나무"

1년의 학습 여정을 **한 그루의 나무가 사계절을 지나는 것**에 비유한다:

```
봄 (1~3개월)        여름 (4~6개월)        가을 (7~9개월)        겨울 (10~12개월)
새싹이 돋아나다      잎이 무성해지다       열매를 맺다           단단해지다

  🌱                   🌳                   🍎                   ❄️ → 🎓
 작은 새싹            무성한 나무           풍성한 열매           졸업, 새로운 시작
```

이 메타포는 Growth Tree 시각화의 핵심이며, 계절에 따라 앱의 미묘한 분위기(배경 색조, 일러스트)도 변화한다.

## 2. 정보 구조 (Information Architecture)

### 2.1 사이트맵

```
MyGraduate
├── Landing Page (/)                    ── 비로그인 시 보이는 소개 페이지
│
├── (Auth)
│   ├── Login (/login)
│   └── Signup (/signup)
│
└── (Main) ── 로그인 후 ──────────────────────────────────────
    │
    ├── Dashboard (/dashboard)           ── 메인 허브 (홈)
    │   ├── 오늘의 상태 카드
    │   ├── 주간 요약 위젯
    │   ├── Growth Tree 미니 뷰
    │   ├── Calendar Heatmap
    │   └── 최근 기록 피드
    │
    ├── 기록 (Record)
    │   ├── Sessions (/sessions)         ── 수업 기록 목록
    │   │   ├── New (/sessions/new)      ── 새 수업 기록
    │   │   └── Detail (/sessions/[id])  ── 기록 상세/편집
    │   └── Quick Capture (/capture)     ── 빠른 메모 (모달 or 별도 페이지)
    │
    ├── 회고 (Reflect)
    │   ├── Reflections (/reflection)         ── 주간 회고 목록
    │   │   └── New (/reflection/new)         ── 새 회고 작성
    │   └── Monthly (/reflection/monthly)     ── 월간 리포트 (Phase 2)
    │
    ├── 성장 (Grow)
    │   └── Growth (/growth)             ── 성장 시각화 허브
    │       ├── Growth Tree 탭
    │       ├── Statistics 탭 (차트들)
    │       ├── Knowledge Map 탭 (Phase 2)
    │       └── Mood Tracker 탭
    │
    ├── 스토리 (Story)
    │   ├── Timeline (/timeline)         ── 연대기 뷰
    │   └── Storybook (/storybook)       ── 졸업 스토리북 (Phase 3)
    │
    ├── 관리
    │   ├── Subjects (/subjects)         ── 과목 관리
    │   └── Settings (/settings)         ── 앱 설정
    │
    └── (Overlay/Modal)
        ├── Quick Capture 모달
        ├── 감정 온도계 피커
        └── Achievement 배지 알림
```

**모바일 하단 네비게이션 세부 사양:**

| 요소 | 크기/값 | 비고 |
|------|--------|------|
| Bottom Tab Bar 높이 | 64px + Safe Area inset | iPhone 홈 인디케이터 대응 |
| FAB 크기 | 56px × 56px | 터치 타겟 최소 48px 초과 |
| FAB 색상 | forest-500 (#2D5A3D) | 그림자: 0 4px 12px rgba(45,90,61,0.4) |
| 탭바 배경 | 반투명 크림 + blur | `backdrop-filter: blur(12px); background: rgba(250,247,242,0.85)` |
| 탭 아이콘 크기 | 24px | 터치 영역: 44px × 44px |
| 활성 탭 색상 | forest-500 | 비활성: warm-gray (#7A7A6E) |

**Quick Capture 바텀 시트 (모바일 전용):**
```
FAB 탭 시 → 바텀 시트 슬라이드 업 (300ms spring)

┌─────────────────────────┐
│    ── (드래그 핸들)       │  ← 4px × 32px, 중앙
│  빠른 메모          ✕    │
│  ─────────────────────   │
│  ┌─────────────────────┐ │
│  │                     │ │  ← 텍스트 영역 (자동 포커스)
│  │  지금 생각을 빠르게   │ │     최소 높이: 120px
│  │  적어보세요...        │ │     폰트: 18px (모바일 가독성)
│  │                     │ │
│  └─────────────────────┘ │
│                          │
│  과목: [자동 선택 ▼]      │
│                          │
│  ┌────┐  ┌──────────────┐│
│  │ 🎤 │  │     저장      ││  ← 저장: 전체 너비, 52px 높이
│  └────┘  └──────────────┘│     🎤: 음성 입력 (Web Speech API)
└─────────────────────────┘
높이: 화면의 55% | 배경 딤: rgba(0,0,0,0.4)
```

**네비게이션 설계 원칙:**
- 핵심 4개 섹션(홈, 기록, 성장, 더보기)만 하단 탭에 배치
- Quick Capture는 항상 접근 가능한 FAB(Floating Action Button)으로
- "더보기"에서 회고, 스토리, 과목, 설정 접근
- Desktop에서는 사이드바에 전체 메뉴 노출
- **모든 터치 타겟 최소 44px × 44px** (Apple HIG 기준)

```
Step 2 (0.3s): 바텀 시트 슬라이드 업 완료
Step 3 (0.3s): 텍스트 필드 자동 포커스 + 키보드 등장
Step 4 (3~5s): 내용 입력 (음성 or 타이핑)
Step 5 (0.1s): "저장" 탭 → 즉시 IndexedDB 저장 → 시트 닫힘
Step 6 (백그라운드): 네트워크 있으면 Supabase 동기화, 없으면 대기

키보드 위 영역 레이아웃:
┌─────────────────────────┐
│ [텍스트 입력 영역]        │
│ ──────────────────────── │
│ 과목: [자동] [🎤] [저장] │  ← 저장 버튼이 키보드 바로 위
└─────────────────────────┘
```

| 키보드 접근 | 모든 인터랙션 키보드 가능 | `tabIndex`, `onKeyDown` |
| 스크린 리더 | 의미 있는 라벨 | `aria-label`, semantic HTML |
| 포커스 표시 | 명확한 포커스 링 | `focus-visible` 스타일 |
| 모션 감소 | 애니메이션 비활성화 옵션 | `prefers-reduced-motion` |
| 폰트 크기 | 사용자 설정 존중 | `rem` 단위 사용 |

## 7. 사용자 경험 세부 사항

### 7.1 첫 사용자 경험 (Onboarding)

```
Step 1: 환영 메시지
"안녕하세요! MyGraduate에 오신 것을 환영합니다.
 1년간의 AI 대학원 여정을, 함께 기록해볼까요?"

Step 2: 기본 정보 입력
- 학기 시작일 / 종료 예정일
- 수업 요일 (금, 토)
- 수강 과목 등록

Step 3: 첫 나무 심기
"당신의 성장 나무를 심었어요! 🌱
 매 수업을 기록할 때마다 이 나무가 자랍니다."

Step 4: 미래의 나에게 편지 (선택)
"1년 뒤의 나에게 편지를 써볼까요?
 졸업할 때 이 편지를 다시 읽게 됩니다."
```

| 주간 회고 | "이번 주 배운 것들을 되돌아볼 시간이에요." | "회고 작성하기" |
